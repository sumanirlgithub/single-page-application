Docker Commands
These are some of the most commonly used Docker commands. Each command has various options and subcommands, so be sure to refer to the Docker documentation for more detailed usage and explanations.


Here's a comprehensive list of Docker commands along with their usage and explanations:

docker run:

Usage: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

Explanation: Used to run a container from a Docker image. It creates a new container instance based on the specified image and executes the specified command within the container.

docker pull:

Usage: docker pull [OPTIONS] NAME[:TAG|@DIGEST]

Explanation: Pulls an image or a repository from a registry to your local machine.

docker build:

Usage: docker build [OPTIONS] PATH | URL | -

Explanation: Builds a Docker image from a Dockerfile and a context directory.

docker push:

Usage: docker push [OPTIONS] NAME[:TAG]

Explanation: Pushes an image or a repository to a registry.

docker images:

Usage: docker images [OPTIONS] [REPOSITORY[:TAG]]

Explanation: Lists all images available on the local machine.

docker ps:

Usage: docker ps [OPTIONS]

Explanation: Lists all running containers.

docker ps -a:

Usage: docker ps -a [OPTIONS]

Explanation: Lists all containers, both running and stopped.

docker start:

Usage: docker start [OPTIONS] CONTAINER [CONTAINER...]

Explanation: Starts one or more stopped containers.

docker stop:

Usage: docker stop [OPTIONS] CONTAINER [CONTAINER...]

Explanation: Stops one or more running containers.

docker rm:

Usage: docker rm [OPTIONS] CONTAINER [CONTAINER...]

Explanation: Removes one or more stopped containers.

docker rmi:

Usage: docker rmi [OPTIONS] IMAGE [IMAGE...]

Explanation: Removes one or more images from the local machine.

docker exec:

Usage: docker exec [OPTIONS] CONTAINER COMMAND [ARG...]

Explanation: Executes a command within a running container.

docker logs:

Usage: docker logs [OPTIONS] CONTAINER

Explanation: Retrieves the logs of a running container.

docker inspect:

Usage: docker inspect [OPTIONS] NAME|ID [NAME|ID...]

Explanation: Returns low-level information about Docker objects such as containers, images, volumes, and networks.

docker network:

Usage: docker network [OPTIONS] COMMAND

Explanation: Manages Docker networks, which allow containers to communicate with each other.

docker volume:

Usage: docker volume [OPTIONS] COMMAND

Explanation: Manages Docker volumes, which are used to persist data generated by and used by Docker containers.

docker-compose:

Usage: docker-compose [OPTIONS] [COMMAND] [ARGS...]

Explanation: Manages multi-container Docker applications using a YAML configuration file called docker-compose.yml.




docker run: Used to run a container from a Docker image, creating a new container instance based on the specified image.
docker build: Builds a Docker image from a Dockerfile and a context directory.
docker exec: Executes a command within a running container.
docker-compose: Manages multi-container Docker applications using a YAML configuration file called docker-compose.yml.
docker ps: Lists all running containers.
docker pull: Pulls an image or a repository from a registry to your local machine.
docker push: Pushes an image or a repository to a registry.
